## Project Title

Optimal Dump Packing — Adaptive Polygon Spot-Point Packing for Autonomous Mine Haul Trucks

---

# 1. PROJECT OVERVIEW

We are building a responsive web-based industrial planning and simulation platform for autonomous mining dump operations.

The core problem is that autonomous mining trucks currently achieve significantly worse dump packing density compared to staffed operations.

Observed:

* Autonomous dump spacing ≈ 7.38m
* Staffed operation spacing ≈ 3.03m
* Approximate density gap ≈ 2.4×

The issue is not truck hardware, sensors, or autonomy quality.

The real issue is:
POOR SPATIAL DUMP PLANNING.

Current systems use:

* fixed rectangular dump spot grids,
* manually adjusted lane planning,
* inefficient edge handling,
* no polygon-aware optimization.

This causes:

* large unused dump regions,
* truck respotting,
* dump edge overshoot,
* low spots/depressions,
* poor dump density,
* unnecessary dozer reshaping.

Our project solves this using:
Adaptive Polygon Spot-Point Packing.

The system dynamically generates the densest safe dumping layout for autonomous haul trucks inside any irregular dump polygon using:

* computational geometry,
* hexagonal close packing,
* rotation optimization,
* lane scheduling,
* truck turning-radius-aware spatial planning.

## Real-World Geospatial Polygon Input

The system supports real-world geospatial dump planning through interactive map-based polygon creation.

Users can define the active dump region using two methods:

### Method 1 — Interactive Polygon Drawing

Users can directly draw the dump boundary on a real map interface by clicking vertices interactively. Once finalized, the polygon is converted into GPS-referenced coordinates and passed to the optimization engine.

### Method 2 — Manual GPS Coordinate Input

Users can manually input GPS coordinates representing the dump boundary. The system automatically renders the polygon on the map and prepares it for optimization.

This allows the system to simulate realistic mine dump environments using real geospatial layouts instead of abstract polygons.

The polygon input layer is visualized using:

* map overlays,
* geospatial polygons,
* real terrain/satellite imagery.

This significantly improves:

# 6. COMPLETE ALGORITHM FLOW

## STEP 1 — Polygon Boundary Ingestion

Input:
GPS-referenced dump polygon.

The polygon represents:
the active dumpable top surface.

Validate:

* polygon simplicity,
* polygon orientation,
* self-intersections.

---

## STEP 2 — Turning Radius Inset Buffer

Each truck requires maneuvering clearance.

Example:
CAT 793 turning radius ≈ 12m.

We shrink the polygon inward using:
polygon inset buffering.

Purpose:

* prevent unsafe edge placements,
* avoid truck overshoot,
* avoid respotting,
* guarantee turning feasibility.

Implementation:
Shapely polygon.buffer(-radius)

Result:
safe dumpable region.

---

## STEP 3 — Polygon Decomposition

Non-convex polygons are decomposed into convex sub-zones.

Purpose:

* improve fillability,
* eliminate dead zones,
* simplify packing operations.

Methods:

* ear clipping,
* Hertel-Mehlhorn decomposition.

---

## STEP 4 — Hexagonal Close Packing

Generate staggered spot arrangement:

Pattern:

* alternating shifted rows,
* honeycomb-like structure.

This is mathematically denser than square grids.

Hex packing efficiency:
≈ 90.7%

Square grid efficiency:
≈ 78.5%

Each generated point represents:
a truck dump spot center.

Important:
we are NOT drawing literal hexagons.
Only the arrangement pattern is hexagonal.

---

## STEP 5 — Rotation Optimization

The ENTIRE generated hexagonal grid is rotated.

We do NOT rotate:

* individual spots,
* individual trucks.

We rotate:
the whole packing structure.

Angles tested:
0° → 60° in 5° increments.

Reason:
different polygon orientations fit different grid angles more efficiently.

Workflow:

* generate grid,
* rotate grid,
* clip against polygon,
* count valid spots,
* choose maximum.

This is controlled brute-force optimization.

---

## STEP 6 — Lane Assignment

Spot points are grouped into approach lanes.

Purpose:
prevent truck conflicts.

Each lane acts as:
a movement corridor.

No two active trucks should occupy the same lane segment simultaneously.

---

## STEP 7 — Dispatch Sequencing

Two sequences exist:

### sequence_in_lane

Determines:
deepest-first ordering within a lane.

Reason:
avoid trucks blocking each other.

### global_sequence

Determines:
overall dispatch order across all lanes.

Round-robin interleaving ensures:
simultaneous truck activity without conflicts.

---

## STEP 8 — Optional Pile Filter

Optional future enhancement.

Checks generated spots against:
existing dump piles.

Unsafe spots are:

* shifted,
* removed,
* or replaced.

This is modular and NOT required for Phase 1.
